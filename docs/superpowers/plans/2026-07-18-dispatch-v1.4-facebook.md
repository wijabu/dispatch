# Dispatch v1.4 — Facebook Marketplace automation — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give Facebook Marketplace the same hands-off automation Dispatch has for OfferUp — post (review-gated), reprice, and relist — driven from the Facebook Android app in the emulator.

**Architecture:** Reuse the entire OfferUp automation stack in `src/automation/android/` (the `Adb` driver, uiautomator parser, device lifecycle, ADBKeyBoard text entry, single Android lock, and the `waitForNode`/`step`/tracker helpers). Add a Facebook selector config, three Facebook flow files, three server actions, UI buttons, and a per-item Facebook category — each the direct analog of an existing OfferUp piece. The Facebook publisher and the drop-to-floor task engine already exist and already carry Facebook's `renew-then-repost` relist policy.

**Tech Stack:** Next.js 16, TypeScript, Drizzle ORM + better-sqlite3, Vitest, Android SDK emulator (`emulator-5554`), ADB + uiautomator, ADBKeyBoard IME.

**Reference implementations to mirror (read these first):** `src/automation/android/flows/post.ts`, `reprice.ts`, `relist.ts` (OfferUp); `src/config/android.ts`; `src/config/offerup-categories.ts`; `docs/offerup-categories.md`; the OfferUp server actions in `src/lib/automation-actions.ts`; the OfferUp buttons in `src/components/ChannelCard.tsx`.

## Global Constraints

- **This is NOT vanilla Next.js** — read the relevant guide in `node_modules/next/dist/docs/` before any routing/server-action change (per `AGENTS.md`).
- **Schema migrations ONLY via `npx drizzle-kit generate` + `migrate`** — never hand-write or hand-apply SQL; journal drift breaks boot.
- **Never wipe or reset `data/`** — Wil's real inventory + the live SQLite DB live there.
- **Timestamps** stored as SQLite `"YYYY-MM-DD HH:MM:SS"` strings (use the existing `toDbDate` helper) — never `toISOString()` into a text-sorted column.
- **Text entry on the emulator uses ADBKeyBoard** via `adb.typeText()` — plain `input text` crashes on non-ASCII (bullets, curly quotes).
- **All Facebook selectors live in `src/config/facebook.ts`** — a Facebook app update is then a one-line edit, never a flow rewrite.
- **Never auto-submit a genuinely new Facebook listing** — the post flow always stops at Facebook's review screen for Wil's tap. Only reprice, the 7-day renew, and the 42-day fresh repost run hands-off.
- **One automation at a time** — reuse the existing `acquireAndroidLock`/`releaseAndroidLock` (shared with OfferUp; the emulator is a single device).
- Emulator serial `emulator-5554`; Facebook package `com.facebook.katana`.
- Branch: `dispatch-v1.4-facebook` (already created; spec committed at `9d3b280`).

### Task 6: Facebook post flow (review-gated)

**Files:**
- Create: `src/automation/android/flows/facebook/post.ts`
- Modify: `src/automation/android/index.ts` (export `postFacebook`)
- Modify: `src/config/facebook.ts` (fill category/condition/next/publish selectors captured here)

**Interfaces:**
- Consumes: `FlowContext`, `AndroidResult`, `step`, `waitForNode`, `newTracker`, `resolveResult` from the shared modules; `facebookSelectors`, `CONDITION_MAP`.
- Produces: `postFacebook(ctx: FlowContext, opts?: { autoSubmit?: boolean }): Promise<AndroidResult>`. Default `autoSubmit: false` → returns `{ status: "posted_review" }`.

- [ ] **Step 1: Capture the remaining composer selectors live**

Drive to the One-item composer and dump: the **category** control and its picker rows, the **Condition** chip nodes, and the **Next / Publish** controls (the flow reaches a review screen then Publish). Add the exact resource-ids/labels to `facebookSelectors`.

- [ ] **Step 2: Implement `postFacebook`**

Mirror `flows/post.ts` (OfferUp), step-by-step under `step(...)`:
1. `launchFacebook`; if `isFacebookLoggedOut` → `{ status: "login_required" }`; `ensureAdbKeyboard`.
2. Push photos to `/sdcard/Pictures` (clear stale `fb_*.jpg` first), media-scan each — identical to OfferUp.
3. Navigate Marketplace → `createListing` → `oneItem`.
4. Attach photos via `addPhotos` (select newest tiles, confirm).
5. Fill `titleField`, `descriptionField`, `priceField` with `adb.typeText(...)`.
6. Select category = `ctx.item.facebookCategory` (+ subcategory if hierarchical) via the captured picker (search-box first if one exists — that pattern beat scroll-and-tap on OfferUp).
7. Select condition = `CONDITION_MAP[ctx.item.condition]` (exact chip-text match).
8. **If `opts.autoSubmit`** (relist repost only): tap Next → Publish, then confirm the post succeeded (Seller Hub shows the new active listing / a "Posted"/approval signal). **Else** (default): stop at the review screen, `foregroundEmulator()`, return `{ status: "posted_review" }` — never tap Publish.
